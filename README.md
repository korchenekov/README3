## Вычислительная характеристика метода последовательного выбора

**Фундаментальные принципы:**
- Стратегия обработки предполагает последовательное выделение минимальных элементов из неупорядоченного сегмента с их последующим размещением в конечной позиции обработанной зоны

**Особенности вычислительного процесса:**
- Внешний итерационный блок выполняет n-1 проходов
- Внутренний цикл сравнений уменьшается от n-1 до 1 операции
- Суммарная вычислительная нагрузка оценивается как квадратичная функция от n

**Асимптотическая оценка: O(n²)**
- Обоснование: Двухуровневая итерационная структура с линейной зависимостью на каждом уровне создает n*(n-1)/2 операций сравнения

## Вычислительная характеристика метода попарного обмена

**Фундаментальные принципы:**
- Стратегия последовательного сравнения и обмена смежных элементов массива
- Многократные проходы по данным до достижения полной упорядоченности

**Особенности вычислительного процесса:**
- Внешний циклический блок выполняет (n-1) полных проходов по массиву
- Внутренний цикл осуществляет (n-i-1) операций сравнения на i-й итерации
- Механизм отслеживания обменов позволяет досрочно завершать обработку

**Детальный анализ циклов:**
- Внешний цикл: for (i = 0; i < n-1; i++) - гарантирует достаточное количество проходов
- Внутренний цикл: for (j = 0; j < n-i-1; j++) - исключает уже упорядоченные конечные элементы
- На каждой итерации внутреннего цикла выполняется сравнение и потенциальный обмен

**Асимптотическая оценка:**
- Верхняя граница: O(n²)
- Нижняя граница: O(n)
- Типичный случай: O(n²)

**Обоснование сложности:**
- Количество сравнений: Σ(i=0 to n-2) (n-i-1) = (n-1) + (n-2) + ... + 1 = n(n-1)/2
- Количество обменов в худшем случае: до 3 × n(n-1)/2 операций (с учетом временной переменной)
- В лучшем случае (уже отсортированный массив): только (n-1) сравнений без обменов
- Квадратичная природа возникает из-за вложенных циклов, каждый из которых зависит от n

## Вычислительная характеристика метода поэлементного включения

**Фундаментальные принципы:**
- Постепенное построение упорядоченной последовательности через корректную вставку элементов
- Каждый новый элемент размещается в соответствующей позиции среди уже обработанных

**Особенности вычислительного процесса:**
- Внешний цикл обрабатывает элементы с индексами от 1 до n-1
- Внутренний цикл осуществляет сдвиги и поиск позиции для вставки
- Текущий элемент сохраняется во временной переменной для последующего размещения

**Детальный анализ циклов:**
- Внешний цикл: for (i = 1; i < n; i++) - начинается со второго элемента
- Внутренний цикл: while (j >= 0 && arr[j] > key) - сдвигает элементы и ищет позицию
- Переменная key = arr[i] сохраняет вставляемый элемент

**Асимптотическая оценка:**
- Верхняя граница: O(n²)
- Нижняя граница: O(n)
- Типичный случай: O(n²)

**Обоснование сложности:**
- Количество сравнений в худшем случае: Σ(i=1 to n-1) i = n(n-1)/2
- Количество присваиваний в худшем случае: Σ(i=1 to n-1) (i + 1) = n(n+1)/2 - 1
- В лучшем случае (уже отсортированный массив): только (n-1) сравнений
- Квадратичная сложность объясняется тем, что каждый новый элемент может потребовать
  просмотра всей уже отсортированной части массива
- Эффективность сильно зависит от начальной упорядоченности данных

## Вычислительная характеристика метода рекурсивного слияния

**Фундаментальные принципы:**
- Рекурсивная декомпозиция на минимальные подмножества с последующим упорядоченным объединением

**Особенности вычислительного процесса:**
- Рекурсивные вызовы создают древовидную структуру глубиной log n
- На каждом уровне рекурсии выполняется слияние всех n элементов
- Циклы слияния работают с линейной сложностью на каждом уровне

**Асимптотическая оценка: O(n log n)**
- Обоснование: Логарифмическое количество уровней рекурсии, на каждом из которых выполняется линейная работа по слиянию

## Вычислительная характеристика метода с переменным шагом

**Фундаментальные принципы:**
- Многоуровневая сортировка с постепенно уменьшающимся расстоянием между сравниваемыми элементами

**Особенности вычислительного процесса:**
- Внешний цикл управляет последовательностью шагов сравнения
- Внутренние циклы выполняют модифицированную сортировку вставками
- Эффективность определяется последовательностью шагов сравнения

**Асимптотическая оценка:**
- Зависит от выбранной последовательности шагов
- Для последовательности n/2^k: O(n²) в худшем случае
- Для последовательности Кнута: O(n^(3/2))
- Обоснование: Количество итераций внутренних циклов зависит от выбранного шага

## Вычислительная характеристика метода с разделением

**Фундаментальные принципы:**
- Стратегия "разделяй и властвуй" с выбором опорного элемента и разделением относительно него

**Особенности вычислительного процесса:**
- Рекурсивные вызовы обрабатывают подмассивы
- Цикл разделения перераспределяет элементы относительно опорного
- Эффективность зависит от сбалансированности разделения

**Асимптотическая оценка:**
- Ожидаемый случай: O(n log n) - при сбалансированном разделении
- Экстремальный случай: O(n²) - при максимально несбалансированном разделении
- Обоснование: Глубина рекурсии варьируется от log n до n

## Вычислительная характеристика метода на основе дерева

**Фундаментальные принципы:**
- Построение специальной древовидной структуры с последующим последовательным извлечением элементов

**Особенности вычислительного процесса:**
- Цикл построения кучи имеет сложность O(n)
- Цикл извлечения элементов выполняет n операций извлечения
- Каждая операция извлечения требует восстановления свойств кучи

**Асимптотическая оценка: O(n log n)**
- Обоснование: n операций извлечения, каждая стоимостью O(log n)

## Вычислительная характеристика прямого поиска

**Фундаментальные принципы:**
- Последовательная проверка элементов до нахождения искомого или завершения просмотра

**Особенности вычислительного процесса:**
- Единый цикл последовательного перебора элементов
- Количество итераций линейно зависит от позиции искомого элемента
- Не требует предварительной подготовки данных

**Асимптотическая оценка:**
- Верхняя граница: O(n) - элемент отсутствует или в конце
- Нижняя граница: O(1) - элемент в начале массива
- Ожидаемый случай: O(n) - в среднем требуется просмотр половины элементов

---

## Вычислительная характеристика дихотомического поиска

**Фундаментальные принципы:**
- Поиск в упорядоченном множестве через последовательное деление области поиска пополам

**Особенности вычислительного процесса:**
- Цикл или рекурсия с геометрическим уменьшением области поиска
- Каждая итерация исключает половину нерелевантных данных
- Требует предварительной сортировки данных

**Асимптотическая оценка: O(log n)**
- Обоснование: Количество итераций равно количеству раз, которое n можно делить на 2

---

## Вычислительная характеристика интерполяционного поиска

**Фундаментальные принципы:**
- Улучшенный поиск в упорядоченных данных с предсказанием позиции на основе распределения значений

**Особенности вычислительного процесса:**
- Цикл поиска использует интерполяционную формулу для оценки позиции
- Эффективен при равномерном распределении данных
- Может деградировать до линейного поиска в неблагоприятных условиях

**Асимптотическая оценка:**
- При равномерном распределении: O(log log n) - двойной логарифм благодаря точному предсказанию позиции
- При неравномерном распределении: O(n) - вырождение до последовательного поиска
- Обоснование: Скорость сходимости зависит от равномерности распределения данных

## Вычислительная характеристика поиска по числам Фибоначчи

**Фундаментальные принципы:**
- Специализированный алгоритм поиска, использующий свойства числовой последовательности Фибоначчи

**Особенности вычислительного процесса:**
- Цикл поиска использует числа Фибоначчи для определения точек разделения
- Каждая итерация уменьшает область поиска в пропорциях золотого сечения
- Требует предварительного вычисления чисел Фибоначчи

**Асимптотическая оценка: O(log n)**
- Обоснование: Количество итераций пропорционально индексу числа Фибоначчи, превышающего n
